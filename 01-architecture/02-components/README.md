# Component Catalog

The canonical roster of EposForge architectural component slots. This
table is the **single source of truth** for component names and their
contract files: prose references them by name (see
[Referencing components](#referencing-components) below), and the
`check-component-links` lint parses the table to resolve and validate
those references.

Components are identified by a **stable descriptive name**, never by a
number. (Earlier drafts used `C`-numbers; they were opaque to readers,
had no decoder, and had drifted out of sync with the contracts — e.g.
Dev Product and Tool Transport were transposed between headings and
citations. Names are self-describing and cannot silently transpose.)

## Referencing components

Reference a component by wrapping its canonical name from the table
above in a Markdown **shortcut reference link** — just the bracketed
name — and let a per-file link-definitions block resolve the path:

```markdown
The [Orchestrator] dispatches each sub-task to a [Dev Product] running
in an [Execution Sandbox].

<!-- component-links (generated by check-component-links.py --write-defs) -->
[Orchestrator]: ./orchestrator.md
[Dev Product]: ./dev-product.md
[Execution Sandbox]: ./execution-sandbox.md
```

In prose the cost is two brackets; the path lives once, at the foot of
the file. This gives readers a self-describing name, gives agents an
unambiguous machine token (the bracket, not a soft capitalization cue),
gives everyone a clickable decoder (the link resolves to the contract),
and lets the lint catch a dead reference the way a bare name never
could.

When you want different surface text, use the **full reference** form so
the bracketed label still matches the table:

```markdown
the [dispatcher][Orchestrator] ... the [sandbox][Execution Sandbox]
```

The definitions block is generated — run
`check-component-links.py --write-defs <file>` after editing, and CI
runs `check-component-links.py --check` to enforce it. See
[Naming Conventions](../../04-standards/01-naming-conventions/naming-conventions.md#component-references).
