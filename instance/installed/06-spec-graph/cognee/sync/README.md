# cognee-sync — Incremental Sync Tool (Phase 1)

Phase 0 of a multi-phase project to replace EposForge's full prune-and-reproject
Cognee ingestion with a git-commit-driven incremental sync. This directory contains
the test harness and HTTP client that will grow into the production sync tool through
Phases 1–5.

---

## What this is

- **Phase 0 (done):** Python project (`uv`-managed, Python 3.13) providing
  `CogneeClient` (`health`, `add_file`, `delete_dataset`), `Config` / `load_config()`,
  and smoke tests for connectivity and dataset lifecycle.

- **Phase 1 (this directory):** Proves a file added to Cognee is extractable and
  queryable. Extends `CogneeClient` with `cognify`, `search`, `list_documents`.
  Integration tests record cognify-implicit/explicit behavior, dedup behavior, and
  search response shapes for Phases 2–4.

- **Phases 2–5 (future):** updatefile/deletefile behavioral tests, ontology stability,
  and the sync tool itself. All build on the client and harness scaffolded here.

## Bootstrap — Windows (primary)

Prerequisites: `uv` installed, age key authorized per the sops-age setup runbook,
and `cognee_api_url` + `cognee_api_token` populated in the central
`secrets.enc.yaml`.

```powershell
# Enable long paths if not already done (one-time, run as Administrator):
Set-ItemProperty -Path "HKLM:\SYSTEM\CurrentControlSet\Control\FileSystem" `
    -Name "LongPathsEnabled" -Value 1

# From repo root:
cd instance\installed\06-spec-graph\cognee\sync
uv sync

# Run smoke tests through the secrets resolver:
python ..\..\..\12-secrets-key-management\bin\epos-secrets `
    uv run pytest -m smoke -v
```

Expected outcomes:
- `uv sync` downloads Python 3.13 if absent and materializes `.venv/` (gitignored).
- `epos-secrets` decrypts `secrets.enc.yaml`, sets `COGNEE_API_URL` and
  `COGNEE_API_TOKEN`, then execs `uv run pytest`.
- `test_api_reachable_and_authenticated` passes against the live `dkr-cgnee-api`
  container.
- `test_dataset_lifecycle_roundtrip` passes: a UUID-named dataset is created and
  then deleted by the fixture teardown. No residual test-prefixed datasets remain
  after the run.

## Secrets

Two new secrets are registered in the central sops-age manifest
(`instance/installed/12-secrets-key-management/sops-age/secrets.toml`):

| Logical name | Runtime env var | Purpose |
|---|---|---|
| `cognee_api_url` | `COGNEE_API_URL` | Base URL of the Cognee HTTP API |
| `cognee_api_token` | `COGNEE_API_TOKEN` | Bearer token for Cognee API auth |

Both are encrypted in `secrets.enc.yaml`. Use `sops secrets.enc.yaml` to add or
rotate values. Neither is committed in plaintext per the AGENTS.md convention.
